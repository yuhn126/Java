JVM은 OS에서 할당받은 메모리 영역(Runtime Data Area)을 세 영역으로 구분한다.

-메소드 영역 : 클래스 코드들이 올라간다.
   -JVM을 시작할 때 생성
   -로딩된 클래스 바이트 코드 내용을 분석 후 저장
   -모든 스레드가 공유

-힙 영역 : 객체가 생성된다.
   -JVM을 시작할 때 생성
   -객체/배열 저장
   -사용되지 않는 객체는 GC(garbage collector)가 자동 제거

-JVM 스택 : 변수들이 생성된다.
   -스레드별로 생성
   -메소드를 호출할 때 마다 Frame을 스택에 추가(push)
   -메소드가 종료하면 Frame을 제거(pop)
